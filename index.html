Dezzy Check Splitter

How to Use
LPT Realty | Accounts Receivable

Dezzy Check Splitter takes a single bulk scan of check packets, automatically detects divider pages, splits the scan into individual transaction files, uses OCR to propose filenames, and outputs individually named PDFs after your review.

Prep

These steps happen outside the tool, but they determine whether processing goes smoothly.

  1. Prep packets: Remove all staples from each check and its related documents.
  2. Insert dividers: Place a red cardstock divider sheet between every packet.
  3. Orient loose checks: Individual checks must be placed vertically (portrait, short edge first) in the tray. Checks are small, so laying them horizontally can drag the page behind them into the feeder and corrupt the pdf.
  4. Stack and scan: Place the full batch upside down in the scanner tray. Execute a single bulk scan. The scanner outputs one large PDF.
Note: After the scan, retrieve each physical check, separate it from its paper documents, and paper clip the documents together. This is unchanged from the current workflow.

Using the Tool

  1. Open Dezzy Check Splitter on your desktop.
  2. Upload the bulk PDF. Drag and drop the file from the bulk scans folder into the upload area, or click to browse.
    1. Optional — Expected Bundles: Before uploading, type how many bundles you expect to find in the scan. If the detected count doesn't match what you typed, a red banner appears on the review screen and the download confirmation will surface the mismatch. Leaving the field blank turns the check off entirely. Use this when you've physically counted the packets and want a safety net against the scanner and the software.
    2. Optional — Paste Deposit Spreadsheet: Click "Optional: Paste Deposit Spreadsheet" on the upload screen and paste the Check # and Property Address columns straight out of Excel. The tool uses this as a lookup table: when it finds a check# in a packet it matches it to your spreadsheet's address rather than relying on OCR alone. Cuts down OCR errors and skips manual entry for the addresses your spreadsheet already knows.
  3. Wait for processing. The tool will detect divider pages, split the PDF into packets, and run OCR on each packet. A progress indicator shows each stage. This typically takes under a minute for a standard batch.
  4. Review the preview screen. Each packet appears as a thumbnail with a proposed filename. Packets that need attention are sorted to the top.
Green — High Confidence
Address detected with high confidence. Quick visual check, then approve.
Yellow — Low Confidence
Address detected but uncertain. Verify the highlighted OCR source before approving.
Red — No Address Found
No address detected. Manually enter the property address before approving. Expected on ~30% of packets.
  1. For each packet, verify the proposed filename. The OCR source is highlighted on the thumbnail so you can confirm where the address was pulled from.
    1. If the filename looks correct: click Approve.
    2. If the filename is wrong: click the filename field to edit it inline, then click Approve.
    3. If the packet shows red (no address found): type the property address in the filename field before approving. This is expected on roughly 30% of packets and is not a tool error.
    4. Zoom: Use the magnifier + / − buttons on the badge row above any preview to confirm the correct address and check# were scanned. The zoom behavior depends on what the tool detected:
      • Standard zoom (one highlight only): + steps you up through 1× → 1.5× → 2× → 3×, recentering on the highlighted region each press and bringing up scrollbars so you can pan.
      • Split zoom (both check# and address detected): A single + click swaps the preview for a stacked split view — the check# zoomed in on the top pane, the address zoomed in on the bottom pane, each at a higher magnification than the single view could reach. Use this when you need to verify both fields at glance-readable size without scrolling. − returns to the 1× full-page view.
Tip: All packets must be approved before you can export. The export button activates once every packet has a confirmed filename.
  1. Click Download ZIP. The tool packages all approved PDFs into a single ZIP file.
  2. Copy the PDFs from the ZIP into the AR SharePoint folder.
  3. Delete the master bulk scan PDF from the bulk scans folder once files are confirmed in SharePoint.

Diagnostic Report (Power Users)

If you want a deeper look at why the tool missed addresses on certain packets — or you're filing a bug report with the dev team — you can export a per-packet diagnostic.

  1. On the review screen, find the … (More Options) button next to Download All as ZIP.
  2. Click it and choose Export Diagnostic Report.
  3. The tool downloads a self-contained .html file you can open in any browser — no extra tooling needed. It includes a summary table (counts of strong / weak / no-match packets, OCR vs. spreadsheet sources, suspicious misses) plus a per-packet breakdown with the raw OCR text, the candidate addresses considered, confidence scores, and the embedded thumbnail. Send it to the dev team alongside the original scan PDF if you're reporting a problem — that combination is usually enough to reproduce and debug almost any miss.

This is a developer tool. Day-to-day reviewers don't need it — you can ignore the … menu entirely and the standard review-and-download flow still works.

File Naming

The tool proposes filenames in this format:

LPTR.[Check #] - [Property Address]

Example: LPTR.1042 - 123 Main St Orlando FL

You can edit any proposed filename inline in the preview screen before approving. The format should match the existing naming convention in SharePoint.

What to Expect

Divider detection 99%+ accuracy on clean scans with properly inserted red dividers.
OCR auto-naming 85%+ of addressable packets auto-named without correction (excludes the ~30% of packets that contain no property address in the documents).
Review time Target under 20 seconds per packet in the preview interface.
Packets with no address Flagged red in the preview. Enter the address manually. This is expected behavior, not a tool error.
